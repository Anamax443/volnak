Princip: dostupnost z veřejných zdrojů, bez obtěžování hráčů

Manažer si do svého izolovaného workspace napíše kapely a jejich sestavy (hráče). Systém stáhne veřejné termíny koncertů, spočítá, kdy je který hráč obsazený, a hlásí kolize — aby nedošlo k dvojímu zabookování téhož muzikanta. Hráč nemusí nic potvrzovat ani napojovat. Facebook/Instagram se nescrapují (ToS + ban účtu) — jen legální kanály.

Datový tok

[veřejné zdroje kapely] Bandzone · vlastní web · Bandsintown/Songkick/Ticketmaster API · ICS │ [konektorová vrstva — adaptéry] víc zdrojů na kapelu, dedupe (datum+místo) ▼ [gigs v D1] termín · místo · zdroj ▼ [výpočet dostupnosti] hráč busy = gig kapely z rosteru manažera, kde hráč hraje ▼ [kolizní přehled] ──► manažer vidí konflikty ve svém workspace [Facebook / Instagram] NEscrapovat ──► legálně: notifikace → e-mail → gmail-mcp · IG Graph business_discovery

plánováno

Identita & viditelnost

Dvourychlostní identita hráče

  • MVP: volné jméno v rosteru (display_name), nula obtěžování hráče
  • F2: ověřená globální osoba (musicians, klíč = e-mail) při self-registraci + kalendáři
  • Matching jmen napříč platformou se v MVP neřeší — žije se na stringech

Model viditelnosti

  • Default: cizí obsazenost jen jako busy, bez detailu (kde/jaká kapela)
  • Detail jen přes visibility_grant (super admin ventil → později souhlas hráče)
  • Platforma nepředává konkurenční přehled zadarmo (chrání pohodlí, ne tajemství)

Rozhodovací log

Přijato

✓Multi-tenant, row-level tenant_id, stavět nad saas-foundation
✓Self-service registrace (magic-link/OTP), super admin jen dohled
✓Dvourychlostní identita (jména → ověřená osoba)
✓Viditelnost default „busy" + visibility_grants
✓Kurátorská ruční vazba hráč↔kapela (ne ze scrapu)
✓Konektorová vrstva, víc zdrojů na kapelu + dedupe
✓MVP na volných jménech — appka jede od první minuty

Zamítnuto

✕Scraping Facebooku/Instagramu (ToS + anti-bot + ban účtu)
✕FB Events přes Graph API — zamčené pro cizí stránky od 2018
✕Headless „čtu vlastní feed jako člověk" — automatizace = ToS problém
✕Globální sestava jako default (nárok na koordinaci manažerů)
＋FB/IG info legálně: notifikace → e-mail → gmail-mcp
